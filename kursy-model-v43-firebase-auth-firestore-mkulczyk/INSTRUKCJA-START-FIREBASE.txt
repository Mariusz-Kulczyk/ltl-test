START — CO ZROBIĆ PO ROZPAKOWANIU

1. Firebase → Firestore Database → Rules
   Wklej zawartość pliku FIREBASE-REGULY-FIRESTORE.txt i kliknij Publish.

2. Firebase → Authentication → Users
   Masz już konto admina: contact@mkulczyk.pl.
   Logujesz się na stronie tym e-mailem i swoim hasłem/kodem z Firebase.

3. Pierwsze logowanie admina
   Wejdź na stronę, kliknij Zaloguj się i zaloguj contact@mkulczyk.pl.
   Po pierwszym wejściu admina aplikacja zapisze podstawowe profile klientów w Firestore.

4. Konta klientów
   W panelu admina możesz edytować profile klientów w „Klienci i konta”.
   Żeby klient mógł się zalogować, musi mieć też użytkownika w Firebase Authentication:
   Authentication → Users → Add user.
   E-mail musi być taki sam jak w profilu klienta.
   Hasło z Authentication jest realnym kodem dostępu.

5. Hosting
   Pliki HTML możesz wrzucić na GitHub Pages.
   Nie używamy npm, mimo że Firebase pokazał przykład npm. W tym pliku użyty jest normalny wariant HTML przez <script type="module">.

UWAGA
Konto i hasło admina nie są zapisane w HTML. Logowanie idzie przez Firebase Authentication.
Dane zapytań, klientów, statusów i archiwum zapisują się w Firestore.
