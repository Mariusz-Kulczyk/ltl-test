START V44

1. Wrzuć pliki z tej paczki do głównego katalogu repozytorium GitHub Pages.
2. W Firebase Authentication logujesz się kontem contact@mkulczyk.pl i swoim hasłem.
3. Firestore Database -> Rules: wklej zawartość FIREBASE-REGULY-FIRESTORE.txt i kliknij Publish.
4. Authentication -> Settings -> Authorized domains: dodaj mariusz-kulczyk.github.io, jeśli logowanie zgłasza błąd domeny.

Poprawka V44:
- po zalogowaniu strona automatycznie przełącza się na panel z tabelą,
- jeśli Firestore chwilowo odrzuci odczyt albo reguły nie są jeszcze wklejone, panel i tak otworzy się na podstawie zalogowanego użytkownika, z lokalną kopią danych.

Konta klientów:
- prawdziwe konto logowania tworzysz w Firebase -> Authentication -> Users -> Add user,
- profil firmy klienta tworzysz/edytujesz w panelu admina na stronie.
